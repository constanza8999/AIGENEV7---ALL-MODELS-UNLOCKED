# 🤝 AIGENEV7 — Contributor Plans & Rewards System

**Free forever. Rewarded by impact.**

AIGENEV7 is and always will be **free**. No subscriptions, no hidden costs, no token limits. But we believe that those who help build the project deserve recognition and rewards. This is our contribution-based plans & rewards system — the more you contribute, the more you unlock.

---

## 📋 How It Works

Every contribution earns you **Contribution Points (CP)**. Points accumulate across your GitHub profile and unlock higher tiers with better rewards.

### Earning Points

| Contribution Type | Points | Examples |
|-------------------|--------|----------|
| 🐛 Bug Report | **5 CP** | Clear bug report with reproduction steps |
| 💡 Feature Suggestion | **5 CP** | Well-described feature with use cases |
| 📖 Documentation | **10 CP** | PR to improve docs, README, or guides |
| ✅ Code Review | **15 CP** | Meaningful review of an open PR |
| 🧪 Testing | **15 CP** | Writing tests, finding edge cases |
| 🔧 Small PR | **25 CP** | Bug fix, typo fix, minor improvement |
| 🚀 Medium PR | **50 CP** | New feature, significant enhancement |
| 💎 Major PR | **100 CP** | Major feature, architecture change |
| 🌍 Translation | **30 CP** | Full translation of docs/UI |
| 👥 Community Support | **10 CP/mo** | Active help in issues & discussions |

## 📜 Contribution Guidelines

### Getting Started

1. **Fork** the repository
2. **Create a branch** (`git checkout -b feature/my-feature`)
3. **Make your changes**
4. **Test** your changes
5. **Commit** with clear messages
6. **Push** and open a Pull Request

### PR Best Practices

- Keep PRs focused on a single change
- Include screenshots for UI changes
- Reference related issues
- Update documentation if needed
- Add tests when possible

### Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Assume good intentions
- No harassment or discrimination
