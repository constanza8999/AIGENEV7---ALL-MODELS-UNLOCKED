🚀 CLI Entry inference-cli.js
Main entry point. Parses commands (ask, chat, models, serve) and routes to the appropriate agents. Also serves the web UI on serve.
ask chat models serve /api/auto-agent /api/infer
🤖 Auto Agent auto-agent.js
Autonomous coding agent with 17 tools. Multi-iteration loop: AI thinks → parses tool calls → executes → returns results → loops until [DONE] or max iterations.
READ WRITE EDIT APPEND INSERT DELETE RENAME COPY REPLACEALL LIST GLOB GREP RUN DIFF FETCH STATUS DONE
🔧 Debug Agent debug-agent.js
Auto-debug loop. Runs a command, captures stderr/stdout on failure, sends to AI for analysis, applies [WRITE]/[EDIT] fixes, retries. Max 5 iterations.
WRITE EDIT EXEC RETRY
🧑‍💻 Chat Agents (30) custom-agents.js
30 predefined agent personas (12 core + 9 defensive + 9 offensive). Each has id, name, emoji, description, and systemPrompt. CRUD operations via CLI commands.
12 Core 🛡️ 9 Defensive ⚔️ 9 Offensive CRUD Export/Import
🧠 Inference Engine inference.js
Shared LLM inference layer. Routes to providers via OpenAI-compatible API. Supports streaming, chunked generation, and automatic provider selection.
infer() Streaming 10 Providers Chunked Gen
🌐 Web UI (Browser) web/
Static HTML/JS/CSS served by the CLI. Chat interface with agent/model selection. Auto Agent panel with streaming log and file change results.

📊 Data Flow Summary

🤖 Auto Agent Flow
/auto "refactor X" → CLI calls runAutoAgent()
→ AI responds with tool calls → parseToolCalls() extracts [READ], [WRITE], [EDIT], ...
→ Safe file operations execute → results formatted → next iteration
→ [DONE] → Returns summary
🔧 Debug Agent Flow
/debug "bun run build" → CLI calls runDebugLoop()
→ Runs command → Fails (exit ≠ 0) → Captures stderr/stdout
→ Sends error to AI → AI returns [WRITE]/[EDIT] fix → Applies fix → Retries
→ If succeeds → ✅ Done. If fails 🤷 → Could not fix
💬 Chat Flow
chat mode → Select agent persona + model → Type message
→ Agent system prompt + conversation history built → infer() called → Streaming response
→ File operations detected in response → User confirms (y/N) → Changes applied
